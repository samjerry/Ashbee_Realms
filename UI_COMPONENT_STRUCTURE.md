## State Behavior

### New Broadcaster (First Visit)
- Header: "Game Setup"
- All fields show defaults
- No yellow highlights
- "Save Changes" button disabled
- No warning message
- No "existing settings" alert

### Existing Broadcaster (Returning)
- Header: "Edit Game Settings"
- All fields show saved values
- No yellow highlights (until changed)
- "Save Changes" button disabled (until changed)
- Shows alert: "You have existing settings. Make changes and save to update."

### After Making Changes
- Changed field labels turn yellow
- "(Modified)" appears next to changed labels
- "Save Changes" button becomes enabled (red)
- "Reset" button appears
- Warning appears: "âš ï¸ You have unsaved changes"
- Browser shows warning if trying to leave

### After Saving
- Success message appears for 3 seconds
- All yellow highlights disappear
- Buttons return to disabled state
- Warning disappears
- Can safely navigate away

### After Clicking Reset
- All fields revert to original values
- Yellow highlights disappear
- Buttons return to disabled state
- Warning disappears

## Color Scheme
- Background: Dark gray (bg-gray-950)
- Cards: Slightly lighter gray (bg-gray-900)
- Primary color: Red (#ef4444 for selected, #dc2626 for buttons)
- Text: White for headers, gray-400 for descriptions
- Warning: Yellow (#facc15)
- Success: Green (#4ade80)
- Error: Red (#f87171)
- Modified labels: Yellow (#facc15)

## Accessibility Features
- Proper label/input associations
- Color + text indicators (not just color)
- Keyboard navigation support
- Browser confirmation for unsaved changes
- Clear error messages
- Loading states with spinner

## Responsive Design
- Mobile: Single column layout
- Tablet: 2-column grid for environment settings
- Desktop: 3-column grid for game modes
- Max width: 4xl (56rem)
- Padding: 1.5rem on all sides
